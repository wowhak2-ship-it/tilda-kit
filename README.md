# Tilda Kit

Свой набор инструментов для Tilda — альтернатива платному ANNEXX.
Панель-конструктор: выбираешь инструмент, настраиваешь ползунками, смотришь живое
превью, копируешь готовый код и вставляешь в Tilda. Код самодостаточный — работает
на сайте всегда, ни от чего не зависит.

Главный принцип (как у ANNEXX): инструменты **модифицируют стандартные блоки Tilda**
(кнопки, формы, меню, табы, галереи, секции) и элементы Zero Block — не заменяют их.
Вставные блоки (слайдер, бургер, маркиза) — дополнение для случаев, когда у Tilda
нет нужного из коробки.

## Инструменты (16)

**Эффекты:** кнопки (5 hover-стилей) · подчёркивания ссылок · плавный скролл ·
появление при скролле · эффекты картинок · анимация цифр · кастомный курсор

**Твики стандартных блоков:** стилизация форм · липкое меню · табы и аккордеон
(T395/T585) · отступы секций · висячие предлоги

**Конструкторы:** слайдер · бургер-меню · бегущая строка · прелоадер

## Как запустить

Из корня репозитория (Casa Alba 1.0):

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .claude/serve.ps1 -Port 8600 -Root tilda-kit
```

Открыть http://localhost:8600/ (панель), либо через preview-конфиг `tilda-kit`.

## Как пользоваться

1. Слева выбери инструмент.
2. Настрой ползунками/цветами — превью справа обновляется сразу.
3. Нажми «Скопировать код».
4. Вставь код в Tilda (см. подсказку под превью — она своя у каждого инструмента).

### Куда вставлять код в Tilda

| Тип | Куда вставлять |
|---|---|
| Эффекты (кнопки, ссылки, скролл, появление) | Настройки сайта → «HTML-код для вставки внутрь HEAD» — действует на весь сайт. Либо блок T123 на одной странице |
| Конструкторы (слайдер, бургер, маркиза, прелоадер) | Блок T123 («HTML-код») в том месте страницы, где должен быть блок |
| Твики (формы, липкое меню) | Настройки сайта → HEAD |

### Поле «Цель: ID или класс»

Эффекты по умолчанию целятся в стандартные классы Tilda (`.t-btn`, `.t-input` и т.п.).
В Tilda своих классов у блоков нет — есть только ID, поэтому поле цели понимает:

| Что вписать | Откуда взять | Во что превратится |
|---|---|---|
| `rec123456789` | ID блока — в настройках блока или в коде страницы | `#rec123456789` (весь блок) |
| `1234567890123456789` (цифры) | Element ID элемента Zero: выдели элемент → Settings → Element ID | `.tn-elem__<ID>` (этот элемент) |
| `my-class` | если класс всё же есть (свой код) | `.my-class` |

Пусто — инструмент работает по стандартным селекторам Tilda.

### Пресеты

Кнопка «Сохранить» под настройками запоминает текущие значения инструмента
(хранятся в браузере). Клик по имени пресета — загрузить, «×» — удалить.

## Файлы

- `index.html` + `css/panel.css` + `js/app.js` — панель (ядро, ничего не знает об инструментах)
- `js/registry.js` — список подключённых инструментов
- `tools/*.js` — инструменты, один файл = один инструмент
- `self-test.html` — авто-проверка всех инструментов (заголовок вкладки: OK/FAIL)
- `tilda-sample.html` — полигон: имитация страницы Tilda со вставленными сниппетами

## Как добавить новый инструмент

1. Скопируй `tools/button-hover.js` как образец.
2. Поменяй `id`, `title`, `category` (`effects` | `constructors` | `tweaks`), `schema`,
   `generate()`, `previewHTML()`.
3. Подключи в `js/registry.js` (import + добавить в массив `tools`).
4. Открой `self-test.html` — все строки должны быть PASS.

## Проверка на реальной Tilda

Селекторы меню для «липкого меню» (`t228, t446, t451, t456, t466, t478`) — стандартные
блоки Tilda; если твой блок меню другой, впиши его класс в поле «CSS-класс меню».

## Этап 2 (план)

Обернуть панель в Chrome-расширение (Manifest V3, content script на tilda.cc):
кнопка прямо в редакторе Tilda, автовставка кода, утилиты админки
(массовая замена URL, поиск по страницам). Модули `tools/*.js` переедут без переделки.
